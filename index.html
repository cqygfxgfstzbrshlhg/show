作业理解

作业说明

本次作业的重点不仅是完成网页，还要理解静态页面从设计、整理到部署上线的完整过程。

我对这次作业的理解

在我看来，这次作业并不只是“把一个页面做出来”，更重要的是学会用前端基础知识去组织信息、 表达内容，并把本地页面真正发布到线上。这样一来，网页不再只是课堂上的练习文件， 而是一份可以公开访问、能够展示学习成果的课程作品。

本次完成的主要内容

  • 根据老师要求梳理页面所需版块，确定展示内容结构。
  • 使用 HTML 搭建页面骨架，保证信息分类清楚、阅读顺序自然。
  • 使用 CSS 完成配色、排版和卡片式布局，让页面更适合作为课程展示页。
  • 进一步了解 GitHub Pages 的发布方式，为后续提交公开链接做好准备。
  • 结合课程作业场景补充说明文字，使页面更像学生本人完成的作品展示。

学习记录

学习过程

从内容整理到页面落地，我主要把这次作业分成了结构搭建、样式调整和部署准备三个步骤。

01

先确定页面该展示什么

一开始我先把老师布置的要求拆开来看，明确哪些内容是必须出现的， 比如基本信息、作业说明、学习过程、成果展示和联系方式。先把内容梳理清楚， 再去设计页面，会比一上来就写样式更有效率。

02

再用 CSS 把页面做得更完整

样式部分我没有刻意追求复杂效果，而是更注意整体观感是否舒服， 比如标题层级是否清晰、卡片之间的间距是否协调、颜色是否清爽， 让老师在浏览时能比较顺畅地看到重点内容。

03

最后整理 GitHub Pages 的部署思路

在完成页面后，我继续了解了 GitHub 仓库上传和 Pages 发布的基本流程， 确保这份作业不仅能在本地打开，也能在部署后生成可访问的网页链接， 方便提交到课程作业平台。

页面设计说明

这份页面的重点不是做成复杂的宣传页，而是尽量做到“老师打开之后能迅速看到重点”。 因此我把内容分成了几个清楚的板块，先交代身份与课程背景，再说明作业理解、学习过程和成果总结， 让整个阅读顺序更自然。

配色上我选择了偏清爽的浅蓝和米白色，整体不夸张，也不会显得太单薄； 版式上采用卡片和分区的方式，既有一定层次感，又符合课程作业展示页应有的稳妥感。

  • 信息集中，老师浏览时能快速找到姓名、学号和课程信息。
  • 文字说明偏自然表达，更接近真实作业汇报，而不是模板式介绍。
  • 项目文件精简，便于直接上传仓库并通过 GitHub Pages 发布。
